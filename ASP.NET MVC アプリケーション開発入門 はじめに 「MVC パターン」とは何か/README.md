<h2 id="02">2. MVC パターンのおさらい</h2>
<p>さて、連載の第 1 回ですが、ひとつ「MVC パターン (MVC アーキテクチャ)」とは何なのかを簡単におさらいしておきましょう。<br>
「MVC パターン」の解説については既に様々な Web サイトの記事があるので、ここでは詳しくは書きません。詳細なところは、MSDN ライブラリを参照したり、Web で「MVC」というキーワードで検索していくつかの記事を読んでみてください。</p>
<ul>
<li><a href="http://msdn.microsoft.com/ja-jp/library/dd381412" target="_blank">[MSDN] ASP.NET MVC の概要</a>
</li></ul>
<p>MVC パターンというものは、Java ならば Struts、PHP ならば CakePHP、Ruby ならば Ruby on Rails という形でパターンが利用されています。その他の言語でも様々な形で利用されています。それぞれの実装の方法は、言語の仕様などにより異なりますが、共通しているものは、Web アプリケーションを Model-View-Controller という 3 つのコンポーネントに分けて開発するという点です。<br>
Web サイトを作る際に、プログラム言語によって得手不得手があるので、それに沿った形で MVC パターンが作られています。</p>
<p>ただし、どの言語でも作っても次の点が共通しています (そこが MVC パターンなのですが)。</p>
<ul>
<li>Model -- データを扱うところ。 </li><li>View -- ユーザーが画面を操作するところ。 </li><li>Controller --ユーザーの操作を Model に伝えるところ。 </li></ul>

<h2 id="03">3. MVC パターンの利点</h2>
<p>MVC パターンの利点について、少し話しておきます。既に Struts や Ruby on Rails などで MVC パターンを使っている方には釈迦に説法のような気もしますが、単なる流行りものではない、ソフトウェア開発の現場で培われた知恵が詰まっていると筆者は思っていますので、ちょっとだけ話させてください。<br>
MVC パターンの最大の利点は、それぞれ担当する領域を分けることにより、コンポーネントごとの機能アップや再帰試験が可能であることだと思います。昨今の Web サイトは大きなものが多く、一人で全てを作成する訳ではありません。複数の人が関わるのが普通になっています。このため、Web サイトの開発を何らかの形で分担しなければいけません。プログラマだけではなく、デザイナーやデータベース管理者などの様々な専門家が関わってきます。</p>
<p>例えば、開発当初では貧弱なビュー (プロトタイプ的なビュー) を作成しておいて、後から専門のデザイナーによるカラフルなビューを作成することが可能です。ビューはモデルが持つデータを参照するだけなので、ビジネス ロジックや入出力の操作ロジックと、分離させてバージョンアップすることができます。<br>
逆に、ビューを固定したままで、モデルやコントローラーの機能を増やすこともできます。モデルが引き渡すデータ構造に変更がなければ、データベースのテーブル構造や、ユーザーが入力するときの検&#35388;機能を後追いで実装することも可能になります。</p>

<h2 id="04">4. MVC パターンのコツ</h2>
<p>MVC パターンを利用するときのコツですが、パターンにとらわれ過ぎないほうが良いでしょう。がちがちに「MVC パターンはこうしないと駄目」というように思い込んでしまうと、どの機能をどこのコンポーネントで実現するのかと悩み過ぎて、前に進めなくなってしまいます。ある意味、小さな規模であればひとまず実装してしまってから動きを確認した後に、リファクタリングを行う程度でも構いません。多人数で作る場合には、どこにどの機能を置くという方向付けとして使っていきます。</p>
<p>例えば、ASP.NET で作るフォーム ベースのアプリケーション (Web フォーム) は手軽にビューが作れること、かつ、コード ビハイドを使ってビューを分離できることに利点があります。うまく作れば、Web フォームでも、MVC パターンと&#20284;たような利点を得ることができます。</p>
<p>ただし、MVC パターンを積極的に使うことで、多人数で Model-View-Controller をどのように分担できるのかという視点が得られます。この視点を使いながら分離していくと、どのコンポーネントに実装を含めるのかが開発していくうちに経験として蓄積されます。</p>
<p>幸いにして、Visual Studio 2010 を使うと、MVC パターンのそれぞれのひな型を作成できます。このひな型に沿ってコードを記述していくことで、保守しやすく癖のないソース コードで揃えられます。</p>
<p>長年続いている先人の知恵を利用/活用して、自分たちの作りたいアプリケーション、実現したい機能を実装してきましょう。</p>
